# Progression and Builds

## Use This For

- 职业、属性、装备、天赋、符文、卡组、成长路线
- 分析 build 是否健康
- 处理 min-max、玻璃炮、万金油、废属性等问题

## Core Tension

成长系统通常在两个目标之间拉扯：

- 专精：强烈风格、明确上限、明显弱点
- 泛用：覆盖场景广、容错高、风格不突出

如果两边都做不到，系统会变成“算数题”；如果两边同时都很强，系统会被单一模板统治。

## Anti-Minmax Levers

- 机会成本：强攻意味着更脆或更短手
- 收益递减：单属性堆太高后回报下降
- 交叉依赖：输出、资源、冷却、控制互相影响
- 遭遇多样性：不同关卡要求不同构筑能力
- 角色职责：不同 build 服务不同场景和团队位置

## Stat System Checklist

- 每个主属性是否都能改变玩家决策
- 是否存在明显废属性
- 单一属性堆叠后是否统治所有场景
- 构筑差异是“玩法差异”还是只是“数值差异”
- 是否支持玩家在中后期纠偏或重构

## Healthy Build Signs

- 有清晰强项，也有真实代价
- 不同玩家会为不同理由选择它
- 不会同时压制新手局和高端局
- 能和遭遇设计形成互动，而不是只比面板

## Common Problems

- 爆发 build 抢走全部价值
- 生存 build 导致所有内容被拖成消耗战
- 控制 build 让敌人失去行动权
- 成长曲线前弱后爆或前爆后废
- 掉落系统只是在给“正确 build”喂资源

## Suggested Output

讨论成长系统时，尽量给出：

- 属性定义
- build archetype 列表
- 关键 trade-off
- 失衡风险
- 第一轮平衡假设
