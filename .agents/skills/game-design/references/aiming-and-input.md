# Aiming and Input

## Scope

用于射击、瞄准、点按、锁定、技能指向和高频交互。

## Core Principle

距离越远、目标越小、运动越快，输入就越难。  
所以“命中感差”不一定是数值问题，可能是输入成本和反馈设计错了。

## What to Check

- 起手位置离目标多远
- 目标判定区是否足够大
- 目标是否在高速移动
- 玩家是否能提前读到轨迹或前摇
- 命中后是否有明确确认反馈

## Device Differences

- 鼠标通常允许更小的目标和更快的精度挑战
- 手柄更依赖锁定、吸附、磁性、较大判定区
- 触屏需要更强容错和更少精密连续输入

## Combat-Specific Questions

- 远程武器的有效射程和目标大小是否匹配
- 快速敌人是否给了足够读招时间
- 玩家 miss 之后是否还能补救
- 头部、弱点、躯干的判定是否服务于玩法而不是只服务于“真实感”

## UI and HUD Questions

- 高频按钮是否靠近常用焦点区域
- 主操作是否足够大且不易误触
- 战斗中关键信息是否在视线中心附近

## Accessibility

- 为反应速度较低的玩家预留容错
- 避免把所有挑战都建立在超高精度上
- 如有必要，允许辅助瞄准、目标锁定或更宽判定
