# Combat Systems

## Use This For

- 设计近战、远程、技能、boss、敌人编队、遭遇战
- 评审“为什么战斗无聊”“为什么 boss 不公平”“为什么某武器独大”
- 输出战斗系统框架或遭遇战草案

## Combat Exchange Model

把一轮交战拆成五段：

1. 读局势
2. 做决策
3. 做承诺
4. 判定结果
5. 进入恢复或连段

战斗深度通常来自“决策”和“承诺”的质量，而不是单纯提高伤害。

## Core Dimensions

### Player Verbs

- 输出：普攻、技能、蓄力、远程、处决
- 防御：闪避、格挡、招架、位移、隐身
- 控制：击退、硬直、打断、牵引、减速

### Pressure Sources

- 伤害压力
- 时间压力
- 空间压力
- 资源压力
- 信息压力

### Counterplay

每个高威胁机制都应该有至少一种可学习的反制方式：

- 读前摇
- 抢节奏
- 拉距离
- 换位
- 资源换资源

### Recovery Windows

玩家和敌人都要有恢复窗口，否则容易出现无限压制或无责任爆发。

## Design Checklist

- 玩家在常态下最常做的 3 个动作是什么
- 战斗的主节奏是爆发、换血、拉扯、控场还是资源博弈
- 是否存在明确的风险回报关系
- 近中远距离是否都有意义
- 新手和高手的上限差异体现在哪
- 玩家失败时能否复盘原因
- 敌人的威胁是否来自不同维度，而不是都只会堆数值

## Boss Questions

- Boss 的威胁是模式识别、输入精度、资源管理，还是站位判断
- 玩家什么时候能安全输出
- 玩家什么时候必须转守
- 二阶段是增加复杂度，还是只是在堆血和堆伤害
- 失败是否因为不懂规则，还是因为执行失误

## Common Failure Modes

- 单一最优武器或 build 覆盖全部场景
- 无限控制链导致对手没有行动权
- 高 burst 让战斗只剩先手判定
- 防御过强使风险消失
- 怪物只靠高血高伤制造“难”
- 反馈不清，玩家命中与受击都不确定

## Output Template

讨论战斗系统时，优先产出：

- 战斗目标体验
- 玩家动词表
- 敌人压力模型
- 资源与冷却规则
- 关键窗口
- 预期失衡点
- 第一轮原型测试项
