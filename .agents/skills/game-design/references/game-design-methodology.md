# Game Design Methodology

## Purpose

把模糊创意收敛成可实现、可测试、可迭代的玩法方案。

## Workflow

### 1. Frame the Experience

- 目标玩家是谁
- 这款游戏的核心幻想是什么
- 玩家为什么愿意重复这一循环
- 成功感来自 mastery、survival、build expression 还是 social dominance

### 2. Define the Verbs

优先写“玩家做什么”，再写“系统是什么”。

- 角色常态动词：移动、观察、锁定、闪避、格挡、连击、蓄力、切换武器
- 高价值动词：打断、处决、控场、爆发、撤退、回复

如果核心动词本身不好玩，后面加再多外围系统都救不回来。

### 3. Separate the Loops

- 核心循环：玩家反复执行的主行为闭环
- 战斗循环：一轮交战中最常见的输入与反馈
- 元循环：升级、解锁、掉落、养成、局外推进

检查三个循环是否互相支撑，而不是互相拖累。

### 4. Write the Failure and Reward Model

- 玩家怎么输
- 输掉后损失什么
- 玩家怎么赢
- 赢了后得到什么
- 奖励是短期强化还是长期成长

### 5. Prototype the Minimum Fun Unit

不要一开始就设计完整大系统。先做“最小可玩的 30 秒”：

- 一个玩家构筑
- 一个敌人或一组敌人
- 一个典型战斗空间
- 一套基本资源规则
- 一组可观察的成功/失败信号

### 6. Iterate With Questions

每轮测试都至少回答：

- 玩家是否理解自己为什么赢/输
- 玩家是否在做有意义的选择
- 玩家是否愿意再次进入循环
- 玩家是否会自发尝试不同策略
- 当前问题是规则问题、数值问题，还是反馈问题

## Deliverables

一个合格的设计输出至少应包含：

- 一句话核心幻想
- 核心循环描述
- 战斗循环描述
- 关键变量列表
- 主要风险
- 下一轮测试目标
