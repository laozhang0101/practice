## 文档目的
这份文档是 M98 战斗框架的导读版，用于在几分钟内统一团队对战斗定位、边界和阅读路径的判断。
- 如果需要完整理解当前战斗设计，请先读 `00A-战斗系统设计总案`。
- 如果需要讨论单个模块的设计细节，再进入对应分文档。
## 一句话定义
M98 的战斗是一个以单人战士塑造为核心、以策略回合感为底层、以 FF7RE 式 `ATB 条 + 暂停行动选技 + 即时技能执行` 为主动交互层、以敌方高威胁动作触发的快反 QTE 为反应层、并保留少量收束 QTE 强化终结的 build 验证战斗系统。
## 本轮新增来源收敛
- 本轮额外吸收了 `原始文档/补充抓取-2026-03-11-外观即性能与0306战斗讨论` 下 4 篇材料。

- `0306` 会议里提到的异步 `PVP`、角色性格延展等方向，继续保留为后续分支，不写入当前单人 `PVE` 主规则。
## 系统定位
- 战斗不只是为了证明玩家手速，而是要证明"我打造的主角真的会这样打"，同时让手动快反形成明确优势。
- 养成负责积累差异，战斗负责把差异打成画面、节奏和高光。
- 战斗还承担把外观预期打实的职责：玩家应先从外观预判角色如何攻击、如何对抗、节奏偏快还是偏稳，再在战斗里被验证。
- 单人战斗是当前核心前提，策略深度主要由单角色构筑承担，而不是阵容搭配承担。
- 常态是 `ATB` 等待期内的自动对峙游走；双方在等待期间自动行走、拉距离、摆姿态，强化对峙表现。
- 任一角色获得行动权时，时间暂停，待玩家或 AI 完成操作后再继续执行。
- 真正的策略选择发生在 `ATB` 满段后的暂停行动中。
- 敌方关键技能会触发快反 QTE；手动操作需要在短窗口内完成对应输入，自动操作则随机选择应对并带有失败概率。
- 手动操作的期望收益和稳定性应明确高于自动操作，才能成立"手动更有优势"。
- 收束 QTE 只保留在终结或角力收束节点，不承担主循环。
## 当前边界
- 当前正文范围只覆盖单人 PVE。
- 单场体验优先验证 1v1 或单 Boss 遭遇，突出主角气质和打法。
- Demo 当前只验证"单角色构筑 + 单场高质量遭遇"是否成立。
## 战斗系统北极星
- 让玩家觉得"这是我心目中的战士"。
- 让一场战斗同时展示搭配、外观、稀有度和个性选择。
- 让低门槛玩家也能打出体面的结果，让高投入玩家能打出明显更个性的结果。
- 让观战者不看面板也能说出这名角色靠什么取胜。
## 当前已定
- 战斗表现优先级高于纯操作深度。
- 核心是单人战士塑造，不是传统队伍阵容配合。
- 半即时、`ATB` 暂停行动和敌方高威胁动作触发的快反 QTE 共同构成主交互。
- 收束 QTE 只允许作为终结强化，不允许取代快反 QTE 和主动落子成为主玩法。
- 手动操作必须在长期收益上明确优于自动操作。
- 长期目标是把"养成结果"稳定转化为"战斗作品感"和"个性展示"。
## 当前核心成功判据
- 测试者能说清楚这名角色"如何进场、如何应对、如何收束"。
- 观察者能指出至少 1 个高光节点和 1 个错误决策点。
- build 差异能通过武器、战甲、身体技能和外观组合被直接看见。
- 不看面板，观察者应能在开场数秒说出主要攻击方式、主要对抗方式和整体节奏倾向。
- 玩家即使不高压输入，也能在一场战斗里做出至少 3 次有后果的判断。
## 统一术语约定
- 战局状态：`中立 / 优势 / 受压`
- 距离层：`近 / 中 / 远`
- 资源层：`ATB 条 / 内在资源`
- 参与层：`常态 / 关键点 / 高光点`
- 决策窗口：`布势与攒条 / 暂停行动 / 快反QTE / 兑现窗口 / 收束窗口`
- 技能来源：`武器 / 战甲 / 身体（内在）`
- 等待状态：`对峙游走`
- 行动状态：`暂停行动`

## 待验证/待补充
- 暂停行动是否需要保留轻度慢时停可访问性版本。
- 武器、战甲、身体三者的长期成长关系。
- 武器、装备、角色、招式四者的长期归属关系。
- 快反 QTE 的目标反应窗和不同 build 的修正范围。
- 统一性能符号如何稳定落到武器、核心部件和灵能结构上。
- 灵能结构如何表达资源与节奏，而不演化成独立第四主玩法。
- 稀有外观、掉落和战斗高光如何进一步绑定.
